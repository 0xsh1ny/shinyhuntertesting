INTRODUCTION TO IoT
What is IoT?

IoT = Internet of Things — a network of physical devices embedded with sensors, software, and connectivity to collect and exchange data over the internet without human intervention.

Examples: Smart AC, fitness band, smart traffic lights, industrial machines.

IoT Security Requirements ⭐ PYQ 5 MarksRemember: AIACIP
  • Authentication — verify device/user identity before access
  • Authorization — control what an authenticated user can do (RBAC)
  • Integrity — data must not be tampered (hash functions)

Challenge areas: Design (low power devices can't run heavy encryption) · Development (no standard framework) · Security (large attack surface)

Software Defined Network (SDN)

Traditional networks: hardware controls routing (rigid). SDN: software controls the network — separates control plane from data plane. In IoT: makes network flexible, programmable, centrally managed. Handles huge number of IoT devices efficiently.

Hidden Terminal vs Exposed Terminal ⭐ PYQ Direct
Hidden Terminal: A ——→ B ←—— C (A & C can't hear each other → both transmit → COLLISION at B) Solution: RTS/CTS mechanism Exposed Terminal: A ←—— B C ——→ D (C hears B, unnecessarily stops transmitting to D — NO collision would happen) Result: Wastage of channel capacity
One line difference: Hidden = collision HAPPENS. Exposed = collision doesn't happen but device still waits unnecessarily.

Data Aggregation & Dissemination

Aggregation = combining data from multiple sensors into one summarized result → saves energy & bandwidth. Techniques: Average, Min, Max, Compression, Filtering.

Dissemination = spreading data/commands FROM base station TO nodes (opposite of aggregation). Example: sending firmware update to all IoT devices.

Virtualization ⭐ PYQ 5 Marks

Virtualization = creating a virtual version of physical resources. A Hypervisor software creates multiple Virtual Machines (VMs) on one physical server.

Advantages:

  • Cost saving — fewer physical machines needed
  • Efficient resource use — no hardware sits idle
  • Easy scaling — spin up new VMs instantly
  • Isolation — one VM crash doesn't affect others
  • Disaster recovery — VMs can be backed up easily
  • Energy saving — fewer physical servers = less power
Cloud Middleware

Middleware = software layer between IoT devices and applications.

Functions: Device management · Data collection · Data processing · Protocol translation · Security (auth + encryption)

Popular platforms: AWS IoT Core · Microsoft Azure IoT Hub · Google Cloud IoT

6LoWPAN ⭐ PYQ Short Note

6LoWPAN = IPv6 over Low Power Wireless Personal Area Networks

IoT devices are small & low power — can't handle full IPv6 packets. 6LoWPAN compresses IPv6 packets to fit small IoT devices. Based on IEEE 802.15.4.

  • Header compression — reduces packet size
  • Packet fragmentation & reassembly
  • Mesh networking support
  • Very low power consumption
  • Used in smart home, industrial IoT, healthcare

Q2(b) — IoT Security Requirements 5 Marks
  • Authentication — every device/user must prove identity before access. Methods: passwords, certificates, biometrics.
  • Authorization — controls what authenticated user can do. Role-based access control (RBAC). Example: sensor can send data but cannot change settings.
  • Data Integrity — data must not be modified during transmission. Achieved using hash functions (SHA, MD5). System detects and rejects tampered data.
  • Confidentiality — data encrypted using TLS/SSL so only intended receiver can read it.
  • Availability — system must always be accessible. Protection against DDoS attacks. Redundant systems ensure no downtime.
  • Privacy — personal user data protected and not misused. Users must have control over what data is collected and shared.

Q6(a) — MAC Protocols: Responsibilities & Design Issues 10 Marks

MAC (Medium Access Control) protocol manages how devices share the wireless channel.

Responsibilities:

  • Channel Allocation — decides which device transmits when. Methods: TDMA (time slots), FDMA (frequency), CSMA (contention).
  • Collision Avoidance — CSMA/CA detects if channel is busy. Uses random backoff timer to retry after collision.
  • Energy Efficiency — puts devices to sleep when not transmitting. Wake up only when needed (duty cycling).
  • Fairness — every node gets equal opportunity to access channel.
  • Reliability — uses acknowledgements (ACK) + retransmits lost packets.

Design Issues:

  • Scalability — must handle thousands of IoT nodes without degrading.
  • Energy Constraint — battery-powered devices need ultra-low power protocols.
  • Mobility — IoT devices may move; protocol must handle changing topology.
  • Heterogeneity — different device types; protocol must work across all.
  • Latency — time-critical apps (healthcare) need low delay.

Q7(a) — Short Note: 6LoWPAN 5 Marks

6LoWPAN = IPv6 over Low Power Wireless Personal Area Networks.

IoT devices are tiny with limited memory & power. Standard IPv6 packets are too large. 6LoWPAN compresses IPv6 headers to fit these devices. Based on IEEE 802.15.4 standard.

  • Header compression — reduces packet size significantly
  • Packet fragmentation & reassembly for large data
  • Mesh networking support
  • Very low power consumption
  • Allows every IoT device to have its own IPv6 address
  • Used in smart homes, healthcare, industrial IoT

Q4(a) — Virtualization + Advantages 5 Marks

Virtualization = creating a virtual version of physical resources (servers, storage, networks). A Hypervisor software runs on physical hardware and creates multiple Virtual Machines (VMs). Each VM acts like an independent computer. One physical server can run 10 virtual servers.

Advantages:

  • Cost saving — fewer physical machines needed
  • Efficient resource use — no hardware sits idle
  • Easy scaling — spin up new VMs instantly
  • Isolation — one VM crash doesn't affect others
  • Disaster recovery — VMs can be backed up easily
  • Energy saving — fewer physical servers = less power
Q4(b) — Processing Overloading + Advantages 5 Marks

Processing Overloading = when IoT system receives more data than it can process in time. Causes: thousands of sensors sending data simultaneously · limited processing power · network congestion.

Advantages of handling it:

  • System stays responsive under heavy load
  • No data loss during peak times
  • Better user experience
  • Improved system reliability
